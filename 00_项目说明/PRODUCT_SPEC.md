# A股短线助手 Product Spec

Version: 0.1
Date: 2026-06-15
Status: Draft

## Product Positioning

「A股短线助手」是一个面向个人使用的 A 股短线研究与复盘工具。

第一版定位为“盘后候选池整理助手”，重点帮助用户在每天收盘后快速完成：

- 整理短线候选股票。
- 解释每只股票进入候选池的原因。
- 标记风险点。
- 生成次日观察条件。
- 保存复盘记录。

本工具只做信息整理、条件筛选和交易计划辅助，不提供确定性收益承诺，也不构成投资建议。

## Version 1 Goal

第一版目标：

每天盘后自动或半自动整理 A 股短线候选池，并为每只候选股票输出关注理由、风险点和次日观察条件。

## Target User

Owner: alexsheng

使用场景：

- 每天盘后复盘。
- 盘前快速查看昨日候选池。
- 对短线强势股、题材股、放量异动股进行归类。
- 记录自己关注、放弃、买入、卖出的理由。

## Core Workflow

1. 导入或获取当天市场数据。
2. 根据筛选规则生成候选股票列表。
3. 为候选股票补充解释信息。
4. 用户查看、标记、删除或补充备注。
5. 生成次日观察清单。
6. 次日或盘后补充实际结果，形成复盘。

## First Version Scope

第一版建议包含以下能力：

- 股票池范围：上证主板 + 深证主板。
- 剔除范围：ST、*ST、创业板、科创板、北交所、B 股和退市整理股票。
- 重点主题：电力股、科技股、商业航天、半导体、PCB 概念。
- 候选池分层：A 池重点关注、B 池观察候补、C 池题材异动记录。
- 候选池列表。
- 股票基础信息：代码、名称、所属市场、所属行业或概念。
- 当日表现：涨跌幅、成交额、换手率、量比、最高价、收盘价。
- 触发原因：涨停、放量、突破、连板、强势板块、异动等。
- 风险提示：高位、缩量、炸板、冲高回落、监管风险、流动性不足等。
- 次日观察条件：开盘强弱、是否放量、是否跌破关键价位、板块是否延续。
- 人工备注。
- 复盘结果：观察成功、失败、放弃、待跟踪。
- 次日验证：明确逻辑成立、逻辑减弱和取消关注的条件。

## Out Of Scope

第一版暂不做：

- 自动下单。
- 账户登录与交易接口。
- 收益承诺或买卖点强推。
- 高频实时盯盘。
- 复杂机器学习预测。
- 付费用户系统。
- 公开社区或多人协作。

## Key Screens

### Dashboard

用途：查看当天候选池概览。

核心信息：

- 今日候选数量。
- 强势板块数量。
- 高风险候选数量。
- 待复盘数量。
- 候选股票列表。

### Candidate Detail

用途：查看单只股票进入候选池的原因。

核心信息：

- 股票代码和名称。
- 当日行情摘要。
- 触发规则。
- 所属板块或概念。
- 风险点。
- 次日观察条件。
- 人工备注。

### Review Log

用途：记录第二天或之后的复盘结果。

核心信息：

- 原始关注理由。
- 实际走势。
- 是否符合观察条件。
- 决策是否合理。
- 可复用经验。

## Data Input

第一版可以先支持 CSV 或手动导入，降低数据源不确定性。

建议字段：

- trade_date
- stock_code
- stock_name
- close

- turnover_rate
- volume_ratio
- high
- low
- open
- industry
- concepts

后续再接入稳定的数据接口。

## Screening Rule Examples

第一版可以从简单、可解释规则开始：

- 先限定股票池：只保留上证主板和深证主板 A 股。
- 剔除 ST、*ST、创业板、科创板、北交所和 B 股。
- 优先保留命中电力、科技、商业航天、半导体、PCB 的股票。
- 当日涨停或接近涨停。
- 成交额显著放大。
- 换手率进入活跃区间。
- 所属板块中多只股票同步走强。
- 突破近期平台或关键均线。
- 连续两日以上明显强于指数。

每条规则都必须能解释“为什么这只股票被选入”。

## Risk Rules

候选股票应同时输出风险标签：

- 高位加速。
- 放量滞涨。
- 冲高回落。
- 炸板。
- 单日过度放量。
- 板块独涨无扩散。
- 流动性不足。
- 近期监管或公告风险。
- 数据缺失或异常。

## Output Format

每只候选股票建议输出：

```text
股票：
代码：
入选原因：
板块/概念：
风险点：
次日观察条件：
人工备注：
复盘结果：
```

## Compliance And Safety

- 所有输出都应表述为辅助分析，不构成投资建议。
- 不承诺收益。
- 不替用户做最终买卖决策。
- 不保存交易账号、密码、Token、Cookie 或 API Key。
- 公开仓库中不得包含敏感配置。
- 数据源、行情时间和更新频率必须清楚标注。

## Success Criteria

第一版完成标准：

- 能导入一份样例行情数据。
- 能生成候选池。
- 每只候选股票都有可解释的入选原因。
- 每只候选股票都有风险标签或明确说明暂无明显风险标签。
- 能保存人工备注。
- 能生成次日观察清单。
- 能记录复盘结果。
