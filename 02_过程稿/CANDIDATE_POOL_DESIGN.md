# Candidate Pool Design

Version: 0.1
Date: 2026-06-15
Status: Draft

## Goal

候选池设计用于把每日筛选结果分成不同跟踪优先级，避免候选股票过多，帮助用户在盘后和次日盘前快速聚焦。

第一版采用 A/B/C 三层候选池：

- A 池：重点关注。
- B 池：观察候补。
- C 池：题材异动记录。

## Design Principles

- 先做排除，再做主题匹配，再做强度判断。
- 先用标签和规则，不急着做复杂评分。
- 每只股票必须有入池原因、风险标签和次日验证条件。
- 主题命中不等于买入理由，只代表关注优先级。
- 候选池不是荐股池，不提供收益承诺或确定性买卖建议。

## Daily Pool Size

建议每日数量上限：

- A 池：最多 5 只。
- B 池：最多 10 只。
- C 池：最多 20 只。

如果候选数量超出上限，按以下优先级保留：

1. 命中重点主题。
2. 板块内多只个股共振。
3. 当日量价强度更高。
4. 风险标签更少。
5. 流动性更好。

当前实现会保留两个口径：

- `规则命中`: 原始规则命中的股票数量。
- `候选数`: 按 A/B/C 每日上限截断后展示在看板中的数量。

超过上限的股票不会进入看板候选池，但会保留 `pool_raw_level`、`candidate_score` 和 `pool_cap_note`，便于后续复盘规则是否过松。

## A Pool

定位：次日重点观察对象。

### Entry Conditions

进入 A 池需同时满足：

- 属于上证主板或深证主板。
- 不属于 ST、*ST、创业板、科创板、北交所、B 股或退市整理股票。
- 命中至少一个重点主题：电力、科技、商业航天、半导体、PCB。
- 当日强度明显，例如涨停、接近涨停、放量突破、连板、板块共振之一。
- 成交额和换手率满足基本活跃度。
- 没有严重风险标签，或风险可被明确说明。

### Typical Tags

- `重点主题`
- `板块共振`
- `涨停`
- `放量`
- `突破`
- `连板`
- `强于指数`

### Required Next-Day Validation

每只 A 池股票必须写明：

- 明天什么情况说明逻辑继续成立。
- 什么情况说明强度减弱。
- 什么情况取消关注。

示例：

```text
成立：高开后承接良好，板块内仍有多只个股走强。
减弱：低开后无法翻红，成交量明显萎缩。
取消：跌破昨日关键支撑，或板块整体退潮。
```

## B Pool

定位：观察候补，适合继续跟踪但不作为最优先。

### Entry Conditions

进入 B 池通常满足：

- 符合股票池范围和排除规则。
- 命中重点主题，或虽未命中重点主题但当日强度突出。
- 有明确异动信号，但强度、板块共振或风险控制不如 A 池。
- 需要等待次日确认。

### Typical Tags

- `观察候补`
- `题材异动`
- `放量`
- `趋势改善`
- `待确认`
- `风险中等`

### Upgrade Conditions

B 池可升级到 A 池，如果次日出现：

- 板块继续扩散。
- 个股放量转强。
- 突破关键压力位。
- 风险标签减少。
- 主题热度明显提升。

### Downgrade Conditions

B 池应降级或移除，如果出现：

- 次日低开低走。
- 板块无延续。
- 放量滞涨。
- 跌破关键支撑。
- 新增明显风险标签。

## C Pool

定位：题材异动记录，不作为重点操作对象。

### Entry Conditions

进入 C 池通常满足：

- 符合股票池范围和排除规则。
- 出现短线异动，但强度、持续性或主题相关度不足。
- 值得记录但暂不重点跟踪。

### Typical Tags

- `异动记录`
- `非重点主题`
- `单股异动`
- `板块未扩散`
- `高风险`
- `等待确认`

### Usage

C 池主要用于：

- 观察是否形成新主线。
- 记录市场边缘信号。
- 复盘题材扩散过程。
- 避免遗漏潜在新方向。

## Hard Exclusion Rules

满足以下任一条件，不能进入 A/B/C 池：

- ST 或 *ST。
- 创业板。
- 科创板。
- 北交所。
- B 股。
- 退市整理或已退市。
- 股票名称异常或代码异常。
- 当日核心行情数据严重缺失。

## Soft Risk Rules

满足以下条件不一定直接剔除，但必须标记风险：

- 成交额过低。
- 换手率异常过高。
- 高位连续加速。
- 放量滞涨。
- 冲高回落。
- 炸板。
- 板块独涨无扩散。
- 单日涨幅过大但无量能支持。
- 近期公告或监管风险。
- 数据源不完整或采集时间异常。

## Theme State

每天建议输出主题状态：

- `主线`: 当日最强、扩散最好、持续性最值得观察的方向。
- `支线`: 有表现但强度弱于主线。
- `退潮`: 前期强势但当日明显走弱的方向。
- `新异动`: 当日刚开始出现集中异动的方向。

第一版重点观察主题：

- 电力股。
- 科技股。
- 商业航天。
- 半导体。
- PCB 概念。

## Candidate Record Format

每只候选股票建议保存：

```text
trade_date:
pool_level: A/B/C
stock_code:
stock_name:
universe:
theme_tags:
market_tags:
entry_reasons:
risk_tags:
next_day_valid_if:
next_day_weak_if:
next_day_remove_if:
manual_note:
review_status:
review_note:
data_source:
captured_at:
```

## Review Status

复盘状态建议使用：

- `待观察`
- `逻辑成立`
- `逻辑减弱`
- `取消关注`
- `误判`
- `继续跟踪`

## Daily Report Structure

每日输出建议：

```text
日期：
数据来源：
采集时间：

市场摘要：
今日主线：
轮动支线：
退潮方向：
新异动方向：

A 池：
B 池：
C 池：

重点风险：
次日观察：
复盘备注：
```

## Strategy Fit

候选池与交易打法的初步匹配：

- A 池：更适合攻击型尾盘突破、主线热点尾盘回流抢筹。
- B 池：更适合均线回踩支撑尾盘企稳、主线热点待确认个股。
- C 池：更适合地量底部首阳、非重点主题异动、信号尚不完整的记录对象。

执行前仍需检查大盘环境、板块状态、个股风险标签和次日验证条件。

## Implementation Priority

第一阶段先实现：

1. 股票池硬过滤。
2. 重点主题标签。
3. A/B/C 池分层。
4. 风险标签。
5. 次日验证字段。
6. 每日报告导出。

后续再实现：

- 主题强度统计。
- 板块扩散度。
- 历史胜率统计。
- 个性化权重。
- 自动复盘分析。
