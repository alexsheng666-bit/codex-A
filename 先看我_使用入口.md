# A股短线助手使用入口

手动使用时，优先双击这四个文件：

1. `1_启动前自检.command`
   - 只检查状态，不刷新、不启动。

2. `2_启动看板.command`
   - 启动当前看板，适合日常快速查看。

3. `3_刷新并启动看板.command`
   - 先刷新数据，再启动看板。

4. `4_关闭看板服务.command`
   - 手动模式下，不看了就双击关闭看板服务。
   - 如果已经安装自动运行，请用 `6_关闭自动运行.command` 停止后台自动运行。

自动运行：

5. `5_安装自动运行.command`
   - 安装后，登录电脑会自动启动看板服务。
   - 周一至周五按交易节奏自动刷新，重点覆盖尾盘窗口。

6. `6_关闭自动运行.command`
   - 不想后台自动启动/自动刷新时，双击关闭自动运行。

7. `7_发布到GitHub Pages.command`
   - 手动把当前看板发布到固定网页链接。
   - 如果普通推送失败，会让你在终端里隐藏输入 GitHub token。
   - 不要把 GitHub token 发到聊天窗口里。

最终推荐模式：

1. 先双击 `4_关闭看板服务.command`，清理之前重复打开的看板服务。
2. 再双击 `5_安装自动运行.command`，安装后台自动运行。
3. 以后电脑开机并登录后，看板会在后台运行。
4. 电脑锁屏不影响访问；电脑睡眠或断网会影响访问。
5. 同一 Wi-Fi 下，用 LAN 地址访问。
6. 人在外面访问，需要再配置 Tailscale 或 Cloudflare Tunnel。
7. 如果使用 GitHub Pages 固定链接，自动刷新后会自动发布到线上。

如果 macOS 弹出是否允许 Python 接收网络连接，请选择“允许”。
如果没有弹出，但窗口显示 `Local:` 或 `LAN:` 地址，说明已经启动成功。

如果固定链接还是旧页面：

1. 先双击 `7_发布到GitHub Pages.command`。
2. 如果窗口提示需要 token，请在终端里粘贴 token；输入时不会显示。
3. 发布成功后等待 1-3 分钟。
4. 浏览器里强制刷新页面，或在链接后加 `?v=当前时间`，例如：

`https://alexsheng666-bit.github.io/codex-A/outputs/stock_report.html?v=20260616`

如果 7 号发布提示无法连接 github.com：

- 这不是 token 问题，说明终端网络连不上 GitHub。
- 先用浏览器打开 `https://github.com/alexsheng666-bit/codex-A`。
- 如果浏览器能打开，可以在 GitHub 网页里手动上传 `outputs/stock_report.html`、`outputs/index.html`、`outputs/publish_metadata.json` 和 `.nojekyll`。
- 如果浏览器也打不开，需要先切换网络或配置代理。

窗口说明：

- `1_启动前自检.command` 跑完后可以关闭。
- `2_启动看板.command` 和 `3_刷新并启动看板.command` 的窗口需要保留，关闭窗口通常会停止服务。
- 不想看窗口时可以最小化到 Dock。
- 手动模式不用看板时，双击 `4_关闭看板服务.command`。
- 自动运行模式不用看板时，双击 `6_关闭自动运行.command`。
- 如果已安装自动运行，平时不需要每天双击 2 或 3。
- 自动刷新需要电脑开机并联网；电脑睡眠时可能会错过当次刷新。

自动刷新时间：

- 周一至周五：9:32、10:30、11:25、13:30、14:15、14:35、14:45、14:50、14:55、15:10、16:10。
- 每次刷新都会重新采集数据、筛选候选股、生成看板，并尝试发布到 GitHub Pages。
- 14:30 以后会更贴近你的尾盘策略观察窗口。
- 盘中或临时需要刷新时，可以打开看板后点右上角“刷新数据”。
