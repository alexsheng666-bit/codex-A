# Screening Rules

Version: 0.1
Date: 2026-06-15
Status: Draft

## Goal

第一版候选池只从沪深主板 A 股中筛选，剔除风险标识和不在目标范围内的股票，再优先关注用户指定的行业与概念方向。

本规则用于后续 CSV 导入、AKShare 采集、字段标准化和候选池生成。

## Stock Universe

### Included

只纳入：

- 上证主板 A 股。
- 深证主板 A 股。

### Excluded

必须剔除：

- ST 股票。
- *ST 股票。
- 创业板股票。
- 科创板股票。
- 北交所股票。
- B 股。
- 退市整理或已退市股票。
- 数据缺失严重、名称异常或停牌时间过长的股票。

## Code-Based First Pass

代码前缀只作为第一层粗筛，最终仍要结合交易所、板块字段和股票名称确认。

### Include Prefixes

建议第一版纳入：

- `600`: 上证主板常见 A 股。
- `601`: 上证主板常见 A 股。
- `603`: 上证主板常见 A 股。
- `605`: 上证主板常见 A 股。
- `000`: 深证主板常见 A 股。
- `001`: 深证主板常见 A 股。
- `002`: 深证主板常见 A 股。
- `003`: 深证主板常见 A 股。

### Exclude Prefixes

建议第一版排除：

- `300`, `301`: 创业板。
- `688`: 科创板。
- `8`, `4`, `9`: 北交所、B 股或其他非目标范围代码，需要结合数据源字段进一步确认。

## Name-Based Exclusion

如果股票简称包含以下内容，直接排除：

- `ST`
- `*ST`
- `S*ST`
- `退`

注意：

- ST 标识可能出现在股票名称不同位置，代码实现时应使用大小写无关匹配。
- 剔除逻辑应在候选池生成前执行，而不是生成后再隐藏。

第一版重点关注以下方向：

## Candidate Priority

第一版候选排序建议：

1. 属于重点主题，且当天明显强于指数。
2. 属于重点主题，且板块内多只个股同步走强。
3. 属于重点主题，且触发涨停、放量、突破或连板规则。
4. 非重点主题但强度极高，可以进入“观察候补”，但默认权重低于重点主题。

## Pool Levels

第一版采用 A/B/C 三层候选池：

- A 池：重点关注，最多 5 只。
- B 池：观察候补，最多 10 只。
- C 池：题材异动记录，最多 20 只。

每只入池股票必须写明：

- 入池原因。
- 风险标签。
- 次日逻辑成立条件。
- 次日逻辑减弱条件。
- 次日取消关注条件。

## Required Output Tags

每只候选股票至少输出：

- `universe`: `沪主板` 或 `深主板`。
- `theme_tags`: 命中的重点主题，例如 `半导体`、`PCB`。
- `exclude_reason`: 如被剔除，记录剔除原因。
- `is_focus_theme`: 是否属于重点主题。
- `screening_reason`: 入选候选池的量价或题材原因。
- `risk_tags`: 风险标签。

## Implementation Notes

- 先过滤股票池，再匹配主题，再计算短线强度。
- 不要用单一关键词决定最终主题，应尽量结合行业、概念、板块和人工确认。
- 同一只股票可以命中多个主题，例如同时属于 `半导体` 和 `PCB`。
- 主题命中只是关注优先级，不等于买入理由。
- 如果数据源没有概念字段，应允许人工维护主题清单作为补充。

## Open Questions

- 科技股是否需要进一步拆成 AI、算力、通信、软件、机器人等子主题？
- 电力股是否更偏传统电力、绿电、电网设备，还是全部覆盖？
- 商业航天是否只看纯题材，还是包含卫星通信、北斗导航、军工航天链？
- PCB 是否只看直接 PCB 公司，还是把覆铜板、载板、上游材料也纳入？
